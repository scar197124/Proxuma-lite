# PROXUMA LITE TRAJECTORY LOCK

## Product Identity
Proxuma Lite is the gateway layer of the Proxuma ecosystem.

It is designed for fast, privacy-first link inspection inside the browser.
It should feel calm, simple, useful, and protective without becoming overwhelming.

## Core Rule
Lite must not compete with Proxuma Shield.

Lite gives quick clarity.
Shield gives deep intelligence.

## Lite Must Include
- Offline browser-based URL scanning
- Simple risk score
- Clear primary concern
- Short explainable findings
- Practical next steps
- QR scanner support
- Privacy-first messaging
- No account, no tracking, no cloud lookup

## Lite Should Avoid
- Deep forensic panels
- Advanced threat intelligence dashboards
- Heavy confidence modeling
- Network reputation checks
- Cloud APIs
- Full redirect crawling
- Multi-stage behavioral simulation
- Professional SOC-style reporting

Those belong in Proxuma Shield.

## Detection Logic Boundary
Lite may detect:
- Suspicious TLDs
- Brand impersonation clues
- Phishing keywords
- Dangerous file extensions
- Encoded redirect clues
- Punycode or odd domain structure
- Basic URL obfuscation

Lite should explain results simply.

Shield may later include:
- Threat classification engine
- Confidence scoring model
- Full technical breakdown
- Redirect-chain expansion
- Reputation intelligence
- Advanced scoring weights
- Exportable reports
- Professional analyst mode

## Current Direction
The next Lite upgrade should focus on:
1. Better wording
2. Cleaner score explanation
3. Clear “Next Steps” section
4. Small detection improvements only
5. Staying fast and understandable

## Design Philosophy
Explain before warning.
Privacy before power.
Clarity before complexity.
Never promise perfect safety.

## Future Chat Instruction
If this file is uploaded in a new chat, continue Proxuma Lite as the simple gateway scanner.
Do not turn it into Shield.
Preserve room for Proxuma Shield as the more powerful scanner.
